# 题目三：FydeOS整体架构分析

这部分就靠AI了

## 一、内核层（Kernel + Boot）

kernel/  
 ├─ binder/             # Android IPC  
 ├─ ashmem/             # Android shared memory  
 ├─ seccomp/            # 系统调用限制  
 ├─ namespaces/         # PID / mount / network namespace  
 ├─ dm-verity/          # verified boot hash check  
 ├─ KVM / virtio/       # Crostini VM 支持  
 └─ drivers/            # GPU, WiFi, camera, OEM hardware

核心调用链：

KVM → crosvm → Linux container → user app  
binder → ARC → Android app

安全与隔离机制：

sandbox → namespaces + seccomp → verified boot

---

## 二、系统服务层（User-space daemon）

platform2/  
 ├─ cryptohome/         # 用户加密 home, TPM 支持  
 ├─ shill/              # 网络管理, DBus  
 ├─ update_engine/      # A/B OTA更新  
 ├─ power_manager/      # 电源管理  
 └─ session_manager/    # 登录 session 管理

关键调用链：

login_manager → cryptohome → session_manager → Chromium desktop  
shill → kernel network driver  
update_engine → partition → reboot

技术栈：

- C++ / DBus / systemd services
    
- TPM / eCryptfs / fscrypt
    

---

## 三、容器 / 虚拟化层（Linux / Android）

crosvm/  
 ├─ devices/            # virtio-net, block, gpu, input  
 ├─ hypervisor/         # KVM interface  
 ├─ vm/                 # memory & sandbox management  
 └─ main.rs             # VM启动主逻辑  
  
vm_tools/crostini/  
 └─ termina VM management scripts  
  
arc/  
 ├─ containers/         # Android container管理  
 ├─ bridge/             # ARC bridge, ChromeOS服务桥接  
 ├─ setup/              # 初始化容器环境  
 └─ services/           # Android服务

调用链示例：

Chrome UI → Crostini manager → crosvm → KVM → kernel → Linux app  
Chrome UI → ARC service → ARC container → Android app → HAL → kernel

技术栈：

- Rust (crosvm)
    
- Go / C++ (ARC)
    
- Linux kernel interfaces (KVM, virtio, binder)
    

---

## 四、UI / 应用层

chrome/  
 ├─ browser/            # browser process  
 ├─ renderer/           # renderer process  
 ├─ gpu/                # GPU process  
 ├─ ui/                 # Ash / Wayland integration  
 └─ mojo/               # IPC interface  
  
ui/ozone/  
 ├─ platform/  
 │   ├─ drm/            # GPU & display  
 │   └─ wayland/        # Wayland backend  
 └─ public/             # API to Chromium

调用链示例：

Web App → Renderer → Mojo → GPU process → Skia / ANGLE → DRM/KMS → Display  
Ash / Wayland → Chromium UI → system services → kernel

技术栈：

- C++ (Chromium core)
    
- Skia / ANGLE / Dawn
    
- Mojo IPC
    
- Wayland / DRM / KMS

## 七、总结原则

整个系统的设计核心：

1. **安全**
    
    - sandbox / seccomp / namespace / dm-verity
        
    - 多进程隔离：renderer / GPU / ARC / Crostini
        
2. **可维护性**
    
    - A/B rootfs update
        
    - read-only rootfs
        
    - clear模块化源码结构
        
3. **应用兼容性**
    
    - Web / Linux / Android 共存
        
    - GPU加速统一接口 (Ozone / Wayland)
        
    - virtio + KVM 支持 Crostini
